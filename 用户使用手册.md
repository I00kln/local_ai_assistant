## 1. 快速入门

│  [系统] 初始化记忆系统...                                       │
│  [系统] 初始化 LLM 客户端...                                    │
│  [系统] 系统就绪，可以开始对话了！                              │
│                                                                 │
│                                                                 │
│                                                                 │
│                                                                 │
│                                                                 │
├─────────────────────────────────────────────────────────────────┤

**界面说明**：

| 区域 | 说明 |
|------|------|
| **状态栏** | 显示当前系统状态和已存储的记忆数量 |
| **聊天区域** | 显示你和AI的对话历史 |
| **输入框** | 在这里输入你想说的话 |
| **控制区** | 各种功能按钮和开关 |

### 1.2 发送第一条消息

**步骤**：

1. 在底部输入框中输入你的消息
2. 按 `Ctrl + Enter` 或点击「发送」按钮
3. 等待AI回复（通常需要几秒钟）

**示例对话**：

```
[你] 你好！我叫张三，我喜欢编程。

[AI] 你好张三！很高兴认识你。我已经记住了你喜欢编程。
     以后如果你有编程相关的问题，随时可以问我！

[记忆] 已存储: "用户名叫张三，喜欢编程"
```

### 1.3 理解回复来源

AI的回复会显示不同的颜色，代表不同的来源：

| 颜色 | 来源 | 说明 |
|------|------|------|
| **蓝色** | 你的消息 | 你发送的内容 |
| **绿色** | 本地AI | 使用本地LLM生成的回复 |
| **紫色** | 云端AI | 使用云端AI（如Gemini）生成的回复 |
| **灰色** | 系统消息 | 系统提示和状态信息 |
| **橙色** | 记忆提示 | 显示检索到的相关记忆 |

**如何判断**：

- 如果回复是**绿色**，说明使用的是本地AI（完全离线）
- 如果回复是**紫色**，说明使用的是云端AI（需要网络）
- 你可以通过勾选/取消「启用云端AI」来切换

## 2. 功能介绍

### 2.1 对话操作

#### 发送消息

**方法1：使用快捷键**
1. 在输入框输入消息
2. 按 `Ctrl + Enter` 发送

**方法2：使用按钮**
1. 在输入框输入消息
2. 点击右侧「发送」按钮

#### 清空对话历史

**操作**：点击「清空对话」按钮

**效果**：
- ✅ 清空当前聊天窗口的显示内容
- ❌ **不会删除**已保存的记忆
- ✅ 保留所有历史记忆数据

**使用场景**：
- 聊天内容太多，想重新开始
- 切换话题，想要一个干净的对话环境

**注意**：清空对话不会影响AI的记忆能力，它仍然记得你之前告诉它的信息。

### 2.2 记忆功能

#### 如何查看已存储的记忆

**方法1：查看系统监控**

1. 点击「系统监控」按钮
2. 系统会显示类似以下信息：

```
=== 系统监控 ===
总记忆数: 127
  - L1 内存层: 25 条 (当前会话)
  - L2 向量库: 52 条 (热数据)
  - L3 数据库: 50 条 (冷数据)

最近记忆:
1. [2026-04-25 10:30] 用户名叫张三，喜欢编程
2. [2026-04-25 10:25] 用户在做一个Python项目
3. [2026-04-25 10:20] 用户偏好使用VS Code编辑器
...
```

**方法2：查看记忆提示**

在对话过程中，系统会自动显示检索到的相关记忆：

```
[你] 我最近在学什么编程语言？

[记忆] 检索到相关记忆:
  - 用户在做一个Python项目
  - 用户偏好使用VS Code编辑器

[AI] 根据我的记忆，你最近在学习Python，并且正在做一个Python项目。
     你还喜欢使用VS Code作为编辑器。需要我帮你解决Python相关的问题吗？
```

#### 记忆的作用

MemoryMind 的记忆系统会：

1. **记住你的偏好**：你喜欢的编辑器、编程语言、工作习惯等
2. **记住重要信息**：你的名字、项目信息、重要日期等
3. **记住对话上下文**：之前讨论过的话题和结论
4. **智能检索**：在合适的时机引用相关记忆

**三层记忆架构**：

```
┌─────────────────────────────────────────────┐
│  L1 内存层（当前会话，约25条）              │
│  • 最快的访问速度                           │
│  • 当前对话的上下文                         │
└─────────────────────────────────────────────┘
              ↓ 自动流转
┌─────────────────────────────────────────────┐
│  L2 向量库（热数据，智能检索）              │
│  • 语义相似度检索                           │
│  • 常用的重要记忆                           │
└─────────────────────────────────────────────┘
              ↓ 自动流转
┌─────────────────────────────────────────────┐
│  L3 数据库（冷数据，长期存储）              │
│  • 所有历史记忆                             │
│  • 权重衰减和自动遗忘                       │
└─────────────────────────────────────────────┘
```

#### 立即保存记忆

**操作**：点击「立即保存记忆」按钮

**作用**：
- 手动触发记忆持久化
- 将内存中的记忆立即写入数据库
- 确保重要信息不会丢失

**使用场景**：
- 输入了重要信息，想立即保存
- 准备关闭程序前，确保数据已保存
- 系统提示保存失败时，手动重试

### 2.3 模式切换

MemoryMind 支持三种运行模式：

#### 仅本地模式

**设置**：
- ☑ 启用本地压缩
- ☐ 启用云端AI

**特点**：
- ✅ 完全离线运行
- ✅ 隐私最佳（数据不上传）
- ✅ 免费使用
- ⚠️ 回复质量取决于本地模型
- ⚠️ 响应速度较慢

**适用场景**：
- 处理敏感信息（密码、密钥等）
- 没有网络连接
- 隐私要求高的场景

#### 仅云端模式

**设置**：
- ☐ 启用本地压缩
- ☑ 启用云端AI

**特点**：
- ✅ 回复质量高
- ✅ 响应速度快
- ⚠️ 需要网络连接
- ⚠️ 需要API密钥
- ⚠️ 可能产生费用

**适用场景**：
- 需要高质量回复
- 复杂问题求解
- 创意写作

#### 混合模式（推荐）

**设置**：
- ☑ 启用本地压缩
- ☑ 启用云端AI

**特点**：
- ✅ 本地处理敏感信息
- ✅ 云端提供高质量回复
- ✅ 平衡隐私和质量
- ✅ 智能降级（云端不可用时自动切换本地）

**适用场景**：
- 日常使用（推荐）
- 需要兼顾隐私和质量

### 2.4 设置说明

#### 显示检索记忆

**开关位置**：控制区的「显示检索记忆」复选框

**作用**：
- ☑ 勾选：在对话中显示AI检索到的相关记忆
- ☐ 取消：不显示记忆检索过程

**建议**：
- 初次使用建议勾选，了解AI如何使用记忆
- 熟悉后可以取消，获得更简洁的对话体验

#### 显示本地结果

**开关位置**：控制区的「显示本地结果」复选框

**作用**：
- ☑ 勾选：显示本地AI的原始回复
- ☐ 取消：只显示最终回复

**建议**：
- 一般用户可以取消勾选
- 开发者或调试时建议勾选

#### 启用本地压缩

**开关位置**：控制区的「启用本地压缩」复选框

**作用**：
- ☑ 勾选：使用本地LLM压缩记忆
- ☐ 取消：禁用压缩功能

**说明**：
- 压缩功能会将长对话压缩成简洁的记忆
- 节省存储空间，提高检索效率
- 建议保持勾选

#### 废话过滤

**开关位置**：控制区的「废话过滤」复选框

**作用**：
- ☑ 勾选：自动过滤无意义的对话（如"好的"、"嗯嗯"）
- ☐ 取消：保存所有对话

**建议**：
- 建议保持勾选
- 减少无效记忆的存储

#### 过滤统计

**操作**：点击「过滤统计」按钮

**作用**：显示废话过滤器的统计数据

**示例输出**：

```
=== 废话过滤统计 ===
总输入: 150 条
已过滤: 23 条 (15.3%)

过滤原因:
  - 规则过滤: 12 条 (太短、纯语气词)
  - 密度过低: 8 条 (信息量不足)
  - 向量匹配: 3 条 (与废话库相似)
```

## 3. 最佳实践

### 3.1 如何让AI更好地记住偏好

#### ✅ 推荐做法

**1. 明确表达偏好**

```
❌ 不好的表达：
"我有时候用Python"

✅ 好的表达：
"我主要使用Python进行开发，这是我最喜欢的编程语言"
```

**2. 提供具体信息**

```
❌ 不好的表达：
"我在做一个项目"

✅ 好的表达：
"我正在开发一个基于Python的Web应用，使用Flask框架"
```

**3. 重复重要信息**

```
你: 我叫张三
AI: 好的，张三！

你: 对了，张三是我的名字，请记住
AI: 我已经记住了，你叫张三！
```

**4. 使用标签和分类**

```
你: 我有一个重要的会议在下周三，这是工作相关的

AI会自动添加标签：
  - 类型: 工作
  - 优先级: 重要
  - 时间: 下周三
```

#### ❌ 避免的做法

**1. 不要说模糊的话**

```
❌ "可能、也许、大概..."
```

**2. 不要频繁改变偏好**

```
❌ 今天说喜欢Python，明天说喜欢Java
   （AI会困惑，不知道哪个是真的）
```

**3. 不要提供矛盾信息**

```
❌ "我不喜欢编程" → "我是个程序员"
   （AI会不知道该信哪个）
```

### 3.2 如何让AI忘记特定信息

#### 方法1：直接告诉AI

```
你: 请忘记我之前说的关于项目X的信息

AI: 好的，我已经删除了关于项目X的记忆。
```

#### 方法2：更新信息

```
你: 我之前说我叫张三，其实我叫李四

AI: 好的，我已经更新了你的名字为李四。
    旧的"张三"记忆会被标记为过期。
```

#### 方法3：清空所有记忆（慎用）

**注意**：目前没有直接的"清空所有记忆"按钮，需要手动操作：

1. 关闭程序
2. 删除以下文件：
   - `memory.db`
   - `chroma_db/` 文件夹
3. 重启程序

**⚠️ 警告**：这将删除所有记忆，不可恢复！

### 3.3 什么情况下应该使用云端模式

#### 推荐使用云端模式的场景

| 场景 | 原因 |
|------|------|
| **复杂问题求解** | 云端AI推理能力更强 |
| **创意写作** | 云端AI语言表达能力更好 |
| **代码调试** | 云端AI代码理解能力更强 |
| **学习新知识** | 云端AI知识库更全面 |
| **多语言对话** | 云端AI语言支持更丰富 |

#### 推荐使用本地模式的场景

| 场景 | 原因 |
|------|------|
| **处理敏感信息** | 数据不上传，隐私最佳 |
| **无网络环境** | 完全离线可用 |
| **日常简单对话** | 本地AI足够应付 |
| **节省费用** | 本地完全免费 |

#### 混合模式的使用建议

**推荐设置**：
- ☑ 启用本地压缩（本地处理敏感信息）
- ☑ 启用云端AI（云端提供高质量回复）

**工作流程**：

```
你输入消息
    ↓
敏感信息过滤器（本地）
    ↓
本地LLM压缩记忆（本地）
    ↓
云端AI生成回复（云端）
    ↓
返回给你
```

**优势**：
- 敏感信息在本地处理，不会上传
- 云端AI提供高质量回复
- 智能降级：云端不可用时自动切换本地

## 4. 常见问题（FAQ）

### Q1：为什么AI回复变慢了？

**可能原因及解决方法**：

| 原因 | 解决方法 |
|------|----------|
| **本地模型太大** | 使用更小的模型（如7B而非13B） |
| **内存不足** | 关闭其他程序，释放内存 |
| **记忆数量过多** | 点击「系统监控」查看记忆数，超过1000条建议清理 |
| **网络问题（云端模式）** | 检查网络连接，或切换到本地模式 |
| **系统负载高** | 重启程序或重启电脑 |

**排查步骤**：

1. 点击「系统监控」查看系统状态
2. 检查记忆数量是否过多（>1000条）
3. 尝试切换到云端模式（如果网络正常）
4. 重启程序

### Q2：如何确认AI记住了我之前说的话？

**方法1：直接询问**

```
你: 你还记得我叫什么名字吗？

AI: 当然记得，你叫张三！
```

**方法2：查看记忆提示**

勾选「显示检索记忆」，在对话中会显示：

```
[记忆] 检索到相关记忆:
  - 用户名叫张三
  - 用户喜欢编程
```

**方法3：查看系统监控**

点击「系统监控」按钮，查看最近的记忆列表。

### Q3：我的对话数据会被上传吗？

**答案**：取决于你使用的模式。

| 模式 | 数据上传情况 |
|------|--------------|
| **仅本地模式** | ❌ 完全不上传，所有数据保存在本地 |
| **仅云端模式** | ✅ 对话内容会上传到云端AI服务器 |
| **混合模式** | ⚠️ 敏感信息本地处理，非敏感内容可能上传 |

**隐私保护措施**：

1. **敏感信息过滤器**：自动识别并保护密码、密钥等敏感信息
2. **本地优先**：敏感信息在本地处理，不会上传
3. **透明提示**：回复颜色标识来源（绿色=本地，紫色=云端）

**建议**：
- 处理敏感信息时，使用仅本地模式
- 日常对话，使用混合模式即可

### Q4：如何备份我的记忆数据？

**方法1：手动备份**

1. 关闭 MemoryMind 程序
2. 复制以下文件到安全位置：
   - `memory.db`（数据库文件）
   - `chroma_db/` 文件夹（向量索引）
   - `.env` 文件（配置文件，可选）
3. 保存到U盘、云盘或其他安全位置

**方法2：使用备份脚本**（如果提供）

```bash
# Linux/macOS
./scripts/backup.sh

# Windows
.\scripts\backup.ps1
```

**备份频率建议**：

| 使用频率 | 备份频率 |
|----------|----------|
| 每天使用 | 每周备份一次 |
| 偶尔使用 | 每月备份一次 |
| 重要信息多 | 每天备份一次 |

### Q5：切换电脑后如何迁移记忆？

**迁移步骤**：

**在旧电脑上**：

1. 关闭 MemoryMind 程序
2. 备份以下文件：
   - `memory.db`
   - `chroma_db/` 文件夹
   - `.env` 文件（包含API密钥等配置）
   - `models/` 文件夹（如果下载了模型）

**在新电脑上**：

1. 安装 MemoryMind（参考安装指南）
2. 将备份的文件复制到新电脑的安装目录
3. 启动 MemoryMind

**验证迁移成功**：

```
你: 你还记得我吗？

AI: 当然记得！你是张三，喜欢编程。
    （说明记忆已成功迁移）
```

**注意事项**：

- 确保两台电脑的操作系统兼容
- 如果路径不同，可能需要修改配置文件
- 首次启动可能需要重建向量索引
