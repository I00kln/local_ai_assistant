## 1. 环境要求与安装

### 1.1 硬件配置要求

#### 最低配置

| 组件 | 要求 | 说明 |
|------|------|------|
| **CPU** | 4 核心 | 支持并发处理和向量计算 |
| **内存** | 8 GB | 运行 LLM + 向量计算 |
| **存储** | 20 GB 可用空间 | 模型 + 数据库 + 日志 |
| **GPU** | 可选 | 加速 LLM 推理（CUDA 支持） |
| **操作系统** | Windows 10+ / macOS 10.15+ / Linux | 跨平台支持 |

#### 推荐配置

| 组件 | 要求 | 说明 |
|------|------|------|
| **CPU** | 8 核心或以上 | 提升并发处理能力 |
| **内存** | 16 GB 或以上 | 支持更大模型和更多记忆 |
| **存储** | 50 GB SSD | 快速向量检索和数据库访问 |
| **GPU** | NVIDIA RTX 3060 或以上 | 显存 8GB+，显著加速推理 |
| **操作系统** | Windows 11 / macOS 12+ / Ubuntu 22.04 | 最新稳定版本 |

#### 不同模型规模的内存需求

| 模型规模 | 参数量 | 最低内存 | 推荐内存 | GPU 显存（可选） |
|----------|--------|----------|----------|------------------|
| 小型模型 | 3B-7B | 8 GB | 12 GB | 6 GB |
| 中型模型 | 7B-13B | 12 GB | 16 GB | 8 GB |
| 大型模型 | 13B-30B | 16 GB | 24 GB | 12 GB |
| 超大模型 | 30B+ | 32 GB | 48 GB | 16 GB+ |

## 2. 配置调优指南

### 2.1 内存配置建议

#### 根据模型大小调整

**config.py 配置项**：

```python
@dataclass
class LocalConfig:
    max_context: int = 8192      # 上下文窗口大小
    max_output_tokens: int = 3000  # 最大输出 token 数
```

**推荐配置**：

| 模型规模 | max_context | max_output_tokens | 说明 |
|----------|-------------|-------------------|------|
| 3B-7B | 4096-8192 | 2000-3000 | 平衡性能与质量 |
| 7B-13B | 8192-16384 | 3000-4000 | 支持更长上下文 |
| 13B-30B | 16384-32768 | 4000-6000 | 高质量长文本 |

**内存估算公式**：
```
所需内存 (GB) ≈ 模型参数 (B) × 1.5 + 向量索引 (GB) + 系统开销 (2GB)
```

#### 向量索引内存优化

```python
@dataclass
class VectorStoreConfig:
    embedding_dimension: int = 512  # 嵌入维度
```

**维度选择建议**：

| 维度 | 内存占用 | 检索质量 | 适用场景 |
|------|----------|----------|----------|
| 384 | 低 | 中等 | 内存受限环境 |
| 512 | 中 | 良好 | 推荐默认值 |
| 768 | 高 | 优秀 | 高精度需求 |

### 2.2 向量索引参数调优

#### ChromaDB 配置优化

```python
@dataclass
class VectorStoreConfig:
    chroma_persist_dir: str = "chroma_db"
    chroma_collection_name: str = "memories"
    embedding_dimension: int = 512
```

**性能调优参数**：

| 参数 | 默认值 | 推荐范围 | 说明 |
|------|--------|----------|------|
| embedding_dimension | 512 | 384-768 | 嵌入向量维度 |
| similarity_threshold | 0.90 | 0.80-0.95 | 相似度阈值 |

#### 检索配置优化

```python
@dataclass
class RetrievalConfig:
    max_retrieve_results: int = 10      # 最大检索结果数
    similarity_threshold: float = 0.90  # 相似度阈值
    l1_min_results: int = 2             # L1 最小结果数
    recall_multiplier: int = 3          # 召回倍数
```

**调优建议**：

| 场景 | max_retrieve_results | similarity_threshold | 说明 |
|------|---------------------|---------------------|------|
| 快速响应 | 5 | 0.85 | 减少检索时间 |
| 平衡模式 | 10 | 0.90 | 推荐默认值 |
| 高召回率 | 20 | 0.80 | 确保不遗漏 |

### 2.3 批处理大小与队列深度设置

#### 异步处理器配置

```python
@dataclass
class AsyncProcessorConfig:
    batch_size: int = 5                    # 批处理大小
    max_queue_size: int = 1000             # 最大队列大小
    queue_full_action: str = "drop_oldest" # 队列满时的动作
    max_pending_compressions: int = 50     # 最大待压缩数
```

**批处理大小调优**：

| 硬件配置 | batch_size | 说明 |
|----------|------------|------|
| 低配（4核8G） | 3-5 | 减少内存压力 |
| 中配（8核16G） | 5-10 | 平衡性能 |
| 高配（16核32G+） | 10-20 | 最大化吞吐量 |

**队列深度设置**：

```python
max_queue_size: int = 1000  # 根据内存调整
queue_full_action: str = "drop_oldest"  # 或 "block" 或 "reject"
```

| 队列策略 | 适用场景 | 说明 |
|----------|----------|------|
| drop_oldest | 高吞吐量 | 丢弃最旧任务，保证实时性 |
| block | 数据完整性 | 阻塞等待，不丢数据 |
| reject | 流量控制 | 拒绝新任务，快速失败 |

### 2.4 压缩与遗忘策略调整

#### 压缩配置

```python
@dataclass
class CompressionConfig:
    min_length: int = 100              # 最小压缩长度
    target_ratio: float = 0.6          # 目标压缩比
    max_segment_length: int = 2000     # 最大分段长度
    fallback_enabled: bool = True      # 启用降级策略
    long_text_threshold: int = 10000   # 长文本阈值
    chunk_size: int = 3000             # 分块大小
```

**压缩策略建议**：

| 记忆类型 | min_length | target_ratio | 说明 |
|----------|------------|--------------|------|
| 短对话 | 50 | 0.7 | 保留更多细节 |
| 标准记忆 | 100 | 0.6 | 推荐默认值 |
| 长文档 | 200 | 0.4 | 激进压缩 |

#### 遗忘策略配置

```python
@dataclass
class DecayConfig:
    batch_size: int = 1000                 # 批处理大小
    decay_rate: float = 0.95               # 衰减率
    min_weight_threshold: float = 0.3      # 最小权重阈值
    forget_age_days: int = 30              # 遗忘天数
    weight_boost_on_access: float = 1.2    # 访问时权重提升
```

**遗忘策略调整**：

| 场景 | decay_rate | forget_age_days | 说明 |
|------|------------|-----------------|------|
| 快速遗忘 | 0.90 | 14 | 减少存储压力 |
| 标准模式 | 0.95 | 30 | 推荐默认值 |
| 长期保留 | 0.98 | 90 | 保留更多历史 |

#### 优雅关闭流程

系统支持优雅关闭，确保数据完整性：

```python
# 系统会自动执行以下步骤：
# 1. 停止接收新请求
# 2. 等待当前请求完成（最多 30 秒）
# 3. 刷新缓冲区到数据库
# 4. 关闭数据库连接
# 5. 保存向量索引
# 6. 退出程序
```

#### 自动备份脚本

**Windows (PowerShell)**：

```powershell
# backup.ps1
$Date = Get-Date -Format "yyyyMMdd_HHmmss"
$BackupDir = "D:\backups\memorymind\$Date"

# 创建备份目录
New-Item -ItemType Directory -Force -Path $BackupDir

# 备份数据库
Copy-Item "memory.db" "$BackupDir\memory.db"

# 备份向量索引
Copy-Item -Recurse "chroma_db" "$BackupDir\chroma_db"

# 备份配置
Copy-Item ".env" "$BackupDir\.env"
Copy-Item "config.py" "$BackupDir\config.py"

# 压缩备份
Compress-Archive -Path $BackupDir -DestinationPath "$BackupDir.zip"

# 清理旧备份（保留最近 30 天）
$OldBackups = Get-ChildItem "D:\backups\memorymind" | 
    Where-Object { $_.LastWriteTime -lt (Get-Date).AddDays(-30) }
$OldBackups | Remove-Item -Recurse -Force

Write-Host "Backup completed: $BackupDir.zip"
```

**Linux/macOS (Bash)**：

```bash
#!/bin/bash
# backup.sh

DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_DIR="/var/backups/memorymind/$DATE"

# 创建备份目录
mkdir -p "$BACKUP_DIR"

# 备份数据库
cp memory.db "$BACKUP_DIR/"

# 备份向量索引
cp -r chroma_db "$BACKUP_DIR/"

# 备份配置
cp .env "$BACKUP_DIR/"
cp config.py "$BACKUP_DIR/"

# 压缩备份
tar -czf "$BACKUP_DIR.tar.gz" -C "$(dirname $BACKUP_DIR)" "$(basename $BACKUP_DIR)"

# 清理旧备份（保留最近 30 天）
find /var/backups/memorymind -type d -mtime +30 -exec rm -rf {} +

echo "Backup completed: $BACKUP_DIR.tar.gz"
```

#### 设置定时备份

**Linux (cron)**：

```bash

```

```bash
tail -f logs/memorymind.log

# 搜索特定时间段日志
grep "2026-04-25 10:" logs/memorymind.log

# 查看性能指标
grep "METRIC" logs/performance.log | tail -50
```

#### 日志清理脚本

```bash
#!/bin/bash
# clean_logs.sh

# 保留最近 7 天的日志
find logs/ -name "*.log" -mtime +7 -exec gzip {} \;

# 移动压缩日志到归档目录
mkdir -p logs/archive
mv logs/*.gz logs/archive/ 2>/dev/null

# 删除 30 天以上的归档
find logs/archive/ -name "*.gz" -mtime +30 -delete

echo "Log cleanup completed"
```

## 4. 监控指标

### 4.1 关键指标列表

#### 系统资源指标

| 指标名称 | 正常范围 | 告警阈值 | 说明 |
|----------|----------|----------|------|
| **CPU 使用率** | 10-60% | > 80% | 持续高 CPU 可能需要扩容 |
| **内存使用率** | 40-70% | > 85% | 接近上限可能 OOM |
| **磁盘使用率** | < 70% | > 85% | 需要清理或扩容 |
| **GPU 显存** | < 80% | > 90% | 模型可能需要卸载 |

#### 应用性能指标

| 指标名称 | 正常范围 | 告警阈值 | 说明 |
|----------|----------|----------|------|

#### 使用自定义脚本

```python
# benchmark.py
import asyncio
import aiohttp
import time
from statistics import mean, median
import numpy as np

class Benchmark:
    def __init__(self, base_url="http://localhost:8000"):
        self.base_url = base_url
        self.results = []
    
    async def request(self, session, endpoint, data):
        start = time.time()
        try:
            async with session.post(
                f"{self.base_url}{endpoint}",
                json=data,
                timeout=aiohttp.ClientTimeout(total=30)
            ) as response:
                await response.json()
                latency = time.time() - start
                self.results.append({
                    "endpoint": endpoint,
                    "latency": latency,
                    "status": response.status
                })
        except Exception as e:
            self.results.append({
                "endpoint": endpoint,
                "latency": time.time() - start,
                "status": 500,
                "error": str(e)
            })
    
    async def run_test(self, num_requests=1000, concurrency=10):
        async with aiohttp.ClientSession() as session:
            tasks = []
            for i in range(num_requests):
                task = self.request(session, "/api/chat", {
                    "message": f"测试消息 {i}",
                    "user_id": f"user_{i % 100}"
                })
                tasks.append(task)
                
                if len(tasks) >= concurrency:
                    await asyncio.gather(*tasks)
                    tasks = []
            
            if tasks:
                await asyncio.gather(*tasks)
    
    def analyze(self):
        latencies = [r["latency"] for r in self.results if "error" not in r]
        errors = [r for r in self.results if "error" in r]
        
        print(f"总请求数: {len(self.results)}")
        print(f"成功请求: {len(latencies)}")
        print(f"失败请求: {len(errors)}")
        print(f"错误率: {len(errors) / len(self.results) * 100:.2f}%")
        print(f"\n延迟统计:")
        print(f"  平均: {mean(latencies):.3f}s")
        print(f"  中位数: {median(latencies):.3f}s")
        print(f"  P95: {np.percentile(latencies, 95):.3f}s")
        print(f"  P99: {np.percentile(latencies, 99):.3f}s")
        print(f"  最大: {max(latencies):.3f}s")
        print(f"  最小: {min(latencies):.3f}s")
        print(f"\n吞吐量: {len(latencies) / sum(latencies):.2f} 请求/秒")

if __name__ == "__main__":
    benchmark = Benchmark()
    asyncio.run(benchmark.run_test(num_requests=1000, concurrency=20))
    benchmark.analyze()
```
